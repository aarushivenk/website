AIRBNB

Redesigning Wishlists

As a design challenge, I redesigned AirBnB wishlists to better allow groups to compare and select the best stay for them.

PURPOSE
Personal Project
TEAM
Solo exploration
TIMELINE
1 week (Sep 2022)
TOOLS
Figma

Overview

Airbnb is a popular short-term rental and experiences app that connects travelers with local places to stay. My friends and I have been using AirBnB frequently as we've planned travel during our semester abroad. I’ve admired Airbnb’s modern and clean design, but there are still ways the app can improve to create a better user experience.

Target Users

College students and young adults planning trips with friends and family

As a college student, Airbnb has been a go-to when trying to plan trips and lower costs. One of the most common frustrations I’m finding is that it’s difficult to narrow down which stay to choose. With so many options, different price ranges to consider, and different sized groups that change at the last moment, it’s tough to get a quick consensus.

How might we help people compare Airbnb stays to find their best fit?

AirBnB Wishlists

In a Wishlist, users can favorite stays they see and add them to a separate wishlist to go back to and share with others.

My friends and I have used wishlists a few times in the past, but we always ended up either:

  • Screenshotting stays
  • Sending links to specific airbnbs, or
  • Putting everything in a Google Sheet because it was difficult to reference a specific stay in a wishlist.

Inspiration: Apple’s Compare Feature

The ability to compare between different models on Apple’s website was a good point of reference as I considered how I could create a comparison tool for Airbnb.

Being able to see features of models side-by-side allows users to see where the differences are. However, this is something a user will have to remember or make note of personally. This tool simply provides users information but doesn't assist them in making the actual evaluation.

The Solution

To combat users from having to use other apps to sort through the airbnbs on their wishlist, I propose adding a “Compare” feature to wishlists.

Users will be able to:

  • Compare custom fields from the airbnbs of their interest,
  • Indicate a favorite airbnb for each category of interest,
  • Cast votes with friends for which airbnbs work the best.

Design: Choosing an orientation

Vertical Layout

  • Clean
  • More whitespace
  • Harder to compare more than 3 stays
  • Vertical and horizontal scrolling needed

Horizontal Layout

  • Tighter comparison
  • Less whitespace
  • Easier to compare fields and stays
  • Focuses users attention on only a couple of stats at a time

I ended up picking the horizontal layout as it is more expandable (allows for more fields to be compared).

Final Design

  • From a wishlist, users can select the “Compare” button at the top of their screen
  • Users can customize which columns are displayed and compare their selected stays
  • Clicking a checkmark badge indicates preference in a column
  • Users can share this table to others or invite them to rank stays
  • Users can scroll horizontally to see more columns, ranking will update in real time

Reflection

Completing this design project made me think about how can we create a solution that helps people out, instead of giving them more work to do.

Moving forward, some next steps are:

  • Usability Tests: Show users the design and see how they would evaluate the stays in their wishlists.
  • Iteration: Continue thinking th rough this idea to see how to better indicate personal and group preferences for stays.
